### üí° Improvement Ideas

**1. Rate Limit Configuration Enhancement**
- Add environment variable `RATE_LIMIT_MAX_READS` and `RATE_LIMIT_MAX_WRITES`
- Default: 300 reads/min, 50 writes/min
- Allow per-environment tuning without code changes
- Consider IP-based exemptions for localhost/127.0.0.1

**2. CMS Health Check Dashboard Widget**
- Add `/api/health` endpoint returning:
  - Server uptime
  - Database file size
  - Last API call timestamps
  - Rate limit usage (requests remaining in current window)
  - File system checks (books/, content/, mission-control/ accessible)
- Display as status card on main dashboard
- Color-coded: green (healthy), yellow (warning), red (error)

**3. Unified Data Layer**
- Create abstraction layer that consolidates:
  - Database.json queries (tasks, agents, messages)
  - File system queries (books, ideas-bank.md)
  - Computed metrics (dashboard stats)
- Single API endpoint `/api/data?resource=books|ideas|tasks|etc`
- Reduces confusion about where data lives
- Makes it easier to migrate to real database later

**4. API Response Caching**
- Cache read-heavy endpoints like `/api/dashboard`, `/api/books`, `/api/ideas`
- TTL: 30-60 seconds (stale data acceptable for most views)
- Dramatically reduce rate limit pressure
- Add cache-busting on write operations
